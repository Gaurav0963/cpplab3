C++ Theory: Classes and Objects

This document explains the fundamental concepts of Object-Oriented Programming (OOP) used in the C++ examples.

What is a Class?

A class is a blueprint for creating objects. It's a user-defined data type that bundles data (variables) and functions (methods) that operate on that data into a single unit. Think of it like the architectural plan for a house. The plan defines the properties of the house (e.g., number of rooms, color of walls) and what you can do with it (e.g., open a door, turn on a light), but it isn't the house itself.

In C++, a class is defined using the class keyword.

class Car { // Members go here };

What is an Object?

An object is an instance of a class. If a class is the blueprint, an object is the actual house built from that blueprint. When you create an object, the program allocates memory to store its specific data. You can create many objects from the same class, and each one will have its own copy of the member variables.

// 'myCar' is an object (an instance) of the 'Car' class. Car myCar;

Access Specifiers: `public` and `private`

Access specifiers are keywords that set the accessibility of class members. This is a core part of encapsulation, which is about bundling data and the methods that operate on that data together, while restricting direct access to some of the object's components.

  • public: Members declared as public can be accessed from anywhere outside the class. Public members form the "interface" of the class—the set of functions that code outside the class can use to interact with the object.
  • private: Members declared as private can only be accessed by other member functions inside the same class. They are completely hidden from the outside world. This protects the object's data from accidental modification. By default, all members of a class are private.

A Clear Example:

This example shows how public members can be accessed directly, while private members cannot.

#include <iostream> #include <string> class Dog { private: // This variable is private. It can only be accessed by member functions of the Dog class. std::string secretDogName; public: // This variable is public. It can be accessed from anywhere, including main(). std::string publicDogBreed; // A public function to set the private name void setSecretName(std::string name) { secretDogName = name; } // A public function that can access the private name void bark() { std::cout << secretDogName << " says: Woof!" << std::endl; } }; int main() { Dog myDog; // Accessing a PUBLIC member is allowed. myDog.publicDogBreed = "Golden Retriever"; std::cout << "The dog's breed is: " << myDog.publicDogBreed << std::endl; // ERROR! You cannot directly access a PRIVATE member from main(). // The line below will cause a compile-time error if you uncomment it. // myDog.secretDogName = "Rex"; // COMPILE ERROR! // You must use a PUBLIC member function to interact with the private member. myDog.setSecretName("Rex"); myDog.bark(); // The public function 'bark' can access the private name. return 0; }

Member Variables and Member Functions

  • Member Variables (Attributes): These are the variables declared inside a class. They represent the data or properties of an object. In the Rectangle class, length and width are member variables.
  • Member Functions (Methods): These are the functions declared inside a class. They define the behavior of an object—the actions it can perform. They typically operate on the member variables of the class. In the Rectangle class, setDimensions() and calculateArea() are member functions.

Getters and Setters: The Gatekeepers of Your Data

Because member variables are private, we need a safe and controlled way to interact with them. We use public "gatekeeper" functions for this.

  • Setter (or Mutator): A public function used to modify the value of a private member variable. Its most important job is to protect the variable by allowing you to add rules or validation logic.
  • Getter (or Accessor): A public function used to retrieve or read the value of a private member variable. It provides safe, read-only access.

A Clear Example:

Imagine we have a Student class and we want to make sure the student's age can never be a negative number.

#include <iostream> #include <string> class Student { private: int age; // This is private. We want to protect it. public: // This is the SETTER. It's our public "gatekeeper" function. void setAge(int newAge) { if (newAge > 0) { age = newAge; } else { std::cout << "Invalid input! Age must be a positive number." << std::endl; } } // This is the GETTER. It's a public "window" to our private data. int getAge() { return age; } }; int main() { Student student1; // ERROR! You cannot do this because 'age' is private. // student1.age = 21; // This line would cause a compile error. // You MUST use the public setter function. std::cout << "Setting age to 21..." << std::endl; student1.setAge(21); std::cout << "The student's age is: " << student1.getAge() << std::endl; std::cout << "\nAttempting to set age to -7..." << std::endl; student1.setAge(-7); // The setter's rule will reject this. std::cout << "The student's age is still: " << student1.getAge() << std::endl; return 0; }

By using a setter (setAge), we gained control over our data and prevented it from becoming corrupted with an invalid value. The getter (getAge) provided a safe way to read the data. This is the core idea of encapsulation.
